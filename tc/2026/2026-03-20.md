#### 2. Technical Decisions & Consensus

- **Topic:** Protocol Version Cutoff Date
  - **Discussion:** The group discussed the timeline for cutting the next
    version of the protocol to include recent breaking changes and new features
    (e.g., cart, catalog, signed totals, order currency).
  - **Decision:** Tentatively set April 3rd as the cutoff date for the next
    version, though this may be adjusted due to upcoming OOO schedules.
  - **Context:** N/A

- **Topic:** Breaking Changes and Versioning
  - **Discussion:** Jing Li raised process questions around handling breaking
    changes, specifically regarding the signed totals change. Debated whether to
    bump the version for individual capabilities versus doing a full
    protocol-level cut. The group noted that Github repository structure
    constraints and the horizontal nature of "totals" make per-capability
    versioning cumbersome at the moment.
  - **Decision:** Go ahead with a full protocol bump for the current totals
    change. Ryan Cheung and Jing Li will investigate better structural options
    (like sub-repos) for capability versioning in the future.
  - **Context:**
    [PR #261 feat: formalize totals contract](https://github.com/Universal-Commerce-Protocol/ucp/pull/261).
