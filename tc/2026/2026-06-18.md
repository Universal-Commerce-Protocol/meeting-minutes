# UCP Tech Council (TC) Meeting - 2026-06-18

**Led by:** Amit Handa

## Attendance

- **Present:** Maxime Najim, Drew Olson, Ilya Grigorik, Lee Richmond, Daniel Wyckoff, Anurag Sinha, Amit Handa, Ryan Cheung, Demian Raspall, James Andersen, Scot DeDeo, Jing Li

## 1. General Topics

- **Topic:** UCP Repository Refactoring for Multi-Vertical Support
    - **Notes:** Jing Li proposed a four-phase plan to refactor the UCP specification repository. The main change involves moving payment constructs from the shopping vertical to common, creating a horizontal structure. The team also discussed SDK releases, debating whether to offer a single combined SDK or vertical-specific packages. Ilya Grigorik advocated for a quarterly release cadence to keep a coherent, shared core protocol.
    - **Context:** [RFC Proposal: Repository Refactoring for Multi-Vertical Support](https://github.com/Universal-Commerce-Protocol/ucp/issues/520)
- **Topic:** Domain Technical Council Nominations
    - **Notes:** Amit Handa announced that nominations are officially open for the newly established Food Ordering Domain Technical Council (DTC). He added that Lodging and Payments charters are expected to be introduced within the next one to two weeks, paving the way for UCP to support multiple verticals.
- **Topic:** Version 08 Planning
    - **Notes:** The team reviewed priorities for the upcoming version 0.8 release, targeting a launch prior to the holiday shopping season. Core targets include 3D Secure, Web Authentication (WBA), and the Fulfillment capability. Jing Li proposed including a location capability, while Amit Handa suggested returns as a best-effort candidate if bandwidth allows.

## 2. Technical Decisions & Consensus

- **Topic:** Structured Fulfillment Capability Schema
    - **Discussion:** Debated whether the fulfillment capability should use a flat object structure or a structured, nested object approach. Amit Handa pointed out that a nested structure aligns with major retail practices, enabling consistent user interface control.
    - **Agreement:** Reached consensus to adopt a structured, nested object approach for the fulfillment capability schema.
    - **Context:** [#507 feat!: fulfillment methods on catalog](https://github.com/Universal-Commerce-Protocol/ucp/pull/507)
- **Topic:** Closure of PR #176
    - **Discussion:** Reviewed the older payment instrument display order proposal (PR #176). The team recognized that ordering arrays is redundant because arrays are ordered by default, whereas ordering maps remains the core technical challenge.
    - **Agreement:** Agreed to close PR #176 in favor of consolidating the map key ordering discussion under PR #525.
    - **Context:** [#176 feat: add optional display_order to payment handlers](https://github.com/Universal-Commerce-Protocol/ucp/pull/176) / [#525 key ordering for UCP map registries](https://github.com/Universal-Commerce-Protocol/ucp/pull/525)

## 3. Open Discussions & TC PR Reviews

- **Topic:** Standardizing Cart Permalinks
    - **Notes:** Ilya Grigorik presented an RFC for cart permalinks to facilitate buyer experiences in contexts where HTTP POST requests are unavailable, such as email campaigns or QR codes. The proposal uses JSON pointers to target schema fields inside a URL. The team agreed that these links should not be cached by intermediate layers as they contain private buyer data. Regarding out-of-stock items, the contract allows businesses to construct the cart to the best of their ability and show a warning rather than failing the request.
    - **Context:** [#523 feat: shopping permalink capability](https://github.com/Universal-Commerce-Protocol/ucp/pull/523)
- **Topic:** Bulk Product Discovery Feeds
    - **Notes:** James Andersen proposed a new RFC for bulk product discovery (product feeds) to solve scalability challenges in agentic commerce. Ilya Grigorik supported the initiative, noting that the team should avoid inventing new formats and instead reuse the existing catalog schema for feed structures.
- **Topic:** Sibling Key Map Ordering
    - **Notes:** Ilya Grigorik presented a proposal to order elements within map structures by utilizing a sibling key to define the sequence. Amit Handa and Drew Olson discussed the implementation, with Amit suggesting the use of UCP annotations rather than "special" keys to prevent technical issues in language-specific generators. The team will continue offline deliberations.
    - **Context:** [#4525 key ordering for UCP map registries](https://github.com/Universal-Commerce-Protocol/ucp/issues/525)
- **Topic:** Additional Meta Fields Extension
    - **Notes:** Reviewed a PR to support extra/meta fields for unstructured data blobs (such as gift wraps or engraving notes), reducing the need for heavyweight custom extensions. Ilya Grigorik emphasized that while useful, this requires a clear contract to define how these dynamic fields are presented. The team will evaluate the PR's readiness for the 0.8 release.
    - **Context:** [#481 feat(shopping): Add additional fields proposal (from Shopify)](https://github.com/Universal-Commerce-Protocol/ucp/pull/481)
