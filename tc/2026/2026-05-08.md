## 1. Technical Discussions & Decisions

- **Topic:** Backporting Policy for Non-Breaking Changes
  - **Discussion:** Discussed establishing a standard operating procedure for backporting pull requests to recent releases.
  - **Decision:** Approved a policy where the default action is to automatically backport non-breaking changes. The burden is now placed on reviewers to explicitly flag reasons why a change should *not* be backported.

- **Topic:** PR and Issue Triage Process
  - **Discussion:** Reviewed a new flowchart for PR life cycles involving manual triage, applying area tags, assigning reviewers based on routing documents, and utilizing status labels (e.g., stale review, blocked).
  - **Decision:** Adopted the manual V1 triage process using status labels to establish clearer accountability. Future automation will be built iteratively based on this deterministic manual process.

- **Topic:** WebMCP Transport Integration
  - **Discussion:** Reviewed a proposal to use WebMCP as a protocol transport. While recognizing the value of agents driving visible page state rather than relying on disparate API calls, technical challenges around UI synchronization were highlighted. An agent's backend calls must trigger storefront events so the user interface reflects the updated state concurrently. 
  - **Decision:** A dedicated deep dive will be scheduled to explore this architecture further.

## 2. Open Discussions & TC PR Reviews

- **Topic:** Loyalty Extension Shape and Scope
  - **Discussion:** Debated simplifying the loyalty response shape by removing the "activated tier" construct to only reflect what is negotiated. Also discussed expanding the implementation scope beyond just checkout to the entire commerce journey (catalog to cart).
  - **Decision:** Agreed to expand the scope back to the broader commerce journey and simplify the data shape. The implicit dependency on the discount extension will be relaxed, though coupling them remains a recommended best practice.
  - **Context:** [#340 feat: loyalty extension for checkout capability](https://github.com/Universal-Commerce-Protocol/ucp/pull/340)

- **Topic:** Identity Linking Backport
  - **Discussion:** Addressed the tension of backporting a breaking schema change for identity linking one month after a release. 
  - **Decision:** TC will open a backport PR alongside a public discussion topic, giving the community a one-week window to voice any strong objections before final approval.
