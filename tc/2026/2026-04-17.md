### UCP Tech Council (TC) Meeting - 2026-04-17
**Led by:** Amit Handa

#### Attendance

#### 1. Technical Decisions & Consensus
- **Topic:** DevOps Issues and PR Labeling
  - **Discussion:** A new `devops` label was introduced for issues and Pull Requests across all repositories to centralize requests for DevOps attention on problems like linter or pipeline failures.
  - **Decision:** Adopted the label to gather insights and improvements from the community for maintainers.

#### 2. Open Discussions

- **Topic:** Terminology Change for Core Concepts (Platform/Business to Client/Provider)
  - **Discussion:** Discussed a massive renaming effort to change "platform" and "business" to "client" and "provider". It was determined that renaming across all internal files, schemas, and file headers would be too invasive and risky.
  - **Recommendation:** Recommendation to stay with the current definitions and focus on clarifying the terminology in documentation instead of a full renaming sweep.
  - **Action:** Tech Council members to review and provide feedback next meeting

- **Topic:** Scopes and Identity Linking Design
  - **Notes:** Debated three scope design variants for identity linking to support fine-grained policies for B2B versus B2C workflows: nested scopes, flat scopes, and config maps. The flat, explicit scope design was generally favored for its simplicity.
  - **Action:** Decision on the design is postponed until early next week to allow for thorough review.
  - **Context:** [#354 feat!: identity linking OAuth 2.0 foundation with capability-driven scopes](https://github.com/Universal-Commerce-Protocol/ucp/pull/354)

- **Topic:** Schema-Validated Documentation Examples
  - **Notes:** Reviewed a proposal to add annotations to in-line examples across all specs, enabling strict schema validation. The goal is to enforce completeness and exactness in documentation.
  - **Action:** Tech Council members to review and provide feedback by next meeting.
  - **Context:** [#359 docs: schema-validated documentation examples](https://github.com/Universal-Commerce-Protocol/ucp/pull/359)

## 3. Review of open Working Draft PRs

- **Topic:** Network Token Credential
  - **Discussion:** Acknowledged that proposed network token functionality can already be supported in the existing token type.
  - **Decision:** The initiative will shift into an improvement in documentation rather than structural schema changes and close this PR.
  - **Context:** [#296 feat: add card network token credential and instrument types](https://github.com/Universal-Commerce-Protocol/ucp/pull/296)

- **Topic:** Attribution Extension
  - **Notes:** Reviewed an attribution extension for checkout sessions. A concern was raised regarding the use of a non-validated `custom` object in the JSON schema, which departs from the preferred extensibility pattern.
  - **Action:** Domain experts will review this construct.
  - **Context:** [#295 feat: add event context extension for cart and checkout sessions](https://github.com/Universal-Commerce-Protocol/ucp/pull/295)

- **Topic:** Extensibility and Compatibility Guidelines
  - **Notes:** Reviewed guidelines on when to use enums and how to build forward-compatible schemas.
  - **Action:** To be reviewed by one additional Tech Council member
  - **Context:** [#290 docs: add extensibility and forward compatibility guidelines](https://github.com/Universal-Commerce-Protocol/ucp/pull/290)

- **Topic:** Returns Extension
  - **Notes:** Acknowledged that returns are an incredibly complicated domain, requiring domain experts to review.
  - **Action:** The council will provide high-level, foundational feedback to ensure the correct domain architecture.
