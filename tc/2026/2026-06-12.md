## 3. Open Discussions & TC PR Reviews

- **Topic:** 3D Secure (3DS) Pressure-Testing Strategy
  - **Discussion:** Discussed the progress of the 3DS deep dive, focusing on protecting the platform from arbitrary content and whether to use an allow list or delegate services to third-party payment service providers to enable easier scaling. The team will pressure-test these concepts against internal implementations.
  - **Notes:** Proposed establishing a set of architectural principles to guide the design of 3DS and other similar actions to validate solutions against a set of constraints. The team agreed that this solves entire classes of problems rather than individual instances. Amit Handa volunteered to structure these principles for review. Discussed the concept of "trusted surfaces" as a way to handle delegated actions, moving away from a "middle manager" model where the platform sits between parties. Suggested that for mechanisms like 3DS, the platform should delegate the action to a trusted party rather than managing the flow directly.
   - **Context:** [#421 feat: add 3DS2 support to spec and schemas](https://github.com/Universal-Commerce-Protocol/ucp/pull/421)
- **Topic:** Missing Response Header Fix
  - **Notes:** Reviewed a fix for a missing consistent response header for signatures in the order operation, and Drew Olson volunteered to review the pull request.
  - **Context:** [#370 fix: add missing signature response headers to get_order](https://github.com/Universal-Commerce-Protocol/ucp/pull/370)
- **Topic:** Payment Instrument Constraints (PR 288/424)
  - **Notes:** Discussed PR 288/424, which aims to generalize payment instrument constraints, such as requiring CVV. Drew Olson advocated for keeping mechanisms simple to avoid creating ad hoc programming languages. Ilya Grigorik added that not all constraints need to be schema-defined and can be expressed in prose.
  - **Context:** [#424 feat: structured constraints for instruments and beyond](https://github.com/Universal-Commerce-Protocol/ucp/pull/424)
- **Topic:** Order Display Ordering (PR 176)
  - **Notes:** Addressed PR 176 regarding order display, noting that introducing ordering to a map structure is a challenging problem. Ilya Grigorik plans to share the considered options and reasoning with the group to see if a more elegant solution exists.
  - **Context:** [#176 feat: add optional display_order to payment handler](https://github.com/Universal-Commerce-Protocol/ucp/pull/176)
- **Topic:** Returns Workflow Review (PR 257)
  - **Notes:** Requested feedback on the returns proposal to ensure it aligns with necessary requirements. Noted the emergence of new European Union regulations regarding returns and the need to accommodate third-party delegation. Greg Smith volunteered to re-review the proposal.
  - **Context:** [#257 feat: add Return Extension to UCP specification](https://github.com/Universal-Commerce-Protocol/ucp/pull/257)
