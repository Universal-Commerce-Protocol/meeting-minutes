## 1. Technical Decisions & Consensus

- **Topic:** Cherry-pick breaking schema fix into recent release
  - **Discussion:** Discussed cherry-picking a time-sensitive breaking schema fix (webhook definition error) into the recently cut release branch. It was noted as an exception given the short time since the release and the low adoption of the current draft.
  - **Decision:** Approved as an exception.

- **Topic:** Core Concepts Terminology Alignment
  - **Discussion:** Discussed the confusion caused by using "platform" and "business" in B2B integration scenarios. Evaluated alternatives such as "consumer" and "client" to make the nomenclature fully agnostic.
  - **Decision:** Approved exploring the terms "client" and "provider". This change should be implemented via a comprehensive search-and-replace across all documentation.

## 2. Open Discussions

- **Topic:** Release Management and Backporting Processes
  - **Notes:** Identified the need for a codified process for backporting non-breaking features, automating workflows, and establishing repository structures for capability versus protocol-level versioning.
  - **Action:** A formal proposal will be drafted as an FAQ document.

- **Topic:** Documentation Generation and Linting Quality of Life
  - **Notes:** Raised concerns over difficulties in building documentation locally and linting failures due to hardcoded dependencies and broken links.
  - **Action:** The technical operations team will be engaged to clean up the repository and implement pre-commit hooks.

## 3. PR Reviews

- **Topic:** Identity Linking (PR #330 / PR #354) (https://github.com/Universal-Commerce-Protocol/ucp/pull/354)
  - **Notes:** The current design needs to be more agnostic and robust to support future changes. It remains a high priority as it currently blocks delegated IDP implementations across the ecosystem.

- **Topic:** Network Tokens (PR #296) (https://github.com/Universal-Commerce-Protocol/ucp/pull/296)
  - **Notes:** Determined that passing Token Requester IDs from client to provider is unnecessary. The PR will pivot towards improving documentation for payment handlers acting as tokenizers.

- **Topic:** Attribution Extension (PR #295) (https://github.com/Universal-Commerce-Protocol/ucp/pull/295)
  - **Notes:** Discussed attaching attribution data to checkouts for reporting. Feedback recommended applying it to the catalog capability and replacing vendor-specific terms with generic identifiers.

- **Topic:** Loyalty Extension (PR #340 / #251) (https://github.com/Universal-Commerce-Protocol/ucp/pull/340)
  - **Notes:** Trimmed down to focus on baseline use cases representing loyalty data in the checkout object when a client is account-linked. Feedback highlighted the need for the protocol to elegantly handle both implicit linking and explicit eligibility signals.
